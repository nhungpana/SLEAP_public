# SLEAP (Lastest Update: 19/04/2026)

Sleep Learning and Explainable Apnea Platform

## Overview

SLEAP is a Streamlit-based research interface for exploring machine learning approaches to obstructive sleep apnea (OSA) risk assessment. The platform combines questionnaire-derived variables, clinical and lifestyle information, reference cohort statistics, CatBoost prediction models, and explainable artificial intelligence methods.
https://58yrdhanodnttej9vsyhsc.streamlit.app/

The current prototype focuses on three main workflows:

1. Individual self-test input with model-based OSA risk prediction.
2. Dataset upload, missing-data inspection, preprocessing, feature selection, and simple CatBoost model training.
3. A chatbot interface for contextualizing model predictions, explanations, and recommendations.

## Scientific Motivation

Obstructive sleep apnea is a common sleep-related breathing disorder associated with intermittent airway obstruction, fragmented sleep, cardiovascular risk, metabolic dysfunction, and impaired daytime functioning. Standard diagnostic evaluation often requires polysomnography or home sleep apnea testing, which may be limited by cost, access, and clinical workflow constraints.

Machine learning models trained on large-scale sleep cohort data may support preliminary risk stratification and help identify clinically relevant predictors. However, predictive tools in health-related contexts should be interpretable, transparent, and carefully contextualized. SLEAP is designed as an experimental interface for studying how prediction, reference population statistics, and model explanations can be presented to users in a more understandable way.

## Core Functions

### 1. Individual Self-Test

The self-test page allows users to enter demographic, anthropometric, comorbidity, sleep habit, OSA symptom, and lifestyle information. The interface then:

- Constructs a model input dataframe from user responses.
- Loads a pre-trained CatBoost model.
- Estimates the probability of moderate sleep apnea risk.
- Displays SHAP-based feature contribution explanations.
- Compares selected user characteristics with distributions from reference sleep cohort data.
- Provides a downloadable PDF report.

### 2. User Dataset Training

The training page allows users to upload a CSV dataset and perform a basic machine learning workflow:

- Inspect dataset shape, row and column counts, duplicated rows, duplicated columns, completed columns, missing-data patterns, numeric columns, categorical columns, and high-cardinality categorical columns.
- Initialize default preprocessing steps, including removing detected ID columns and columns with more than 50 percent missing data.
- Add, remove, reorder, and apply preprocessing steps.
- Remove columns by missing-data threshold or manual selection.
- Replace missing values with user-defined values, including per-column values for mixed numeric and categorical selections.
- Fill numeric missing values with maximum, minimum, median, or average values.
- Run a candidate-label check that tests low-cardinality columns with quick logistic regression models, excludes other candidate-label columns from the features, and flags perfect or near-perfect candidates that may represent easy labels or possible target leakage.
- Automatically move flagged candidate-label columns to unselected features while allowing users to add them back manually.
- Select a binary label column and model features.
- Train a simple CatBoost classifier.
- Review sensitivity, specificity, negative predictive value, F1 score, Matthews correlation coefficient, AUC, confusion matrix, and ROC curve.
- Download the trained CatBoost model file.

### 3. Chatbot Explanation Interface (On going)

The chatbot page is intended to help contextualize predictions and explanations. Its role is to support interpretation of model outputs, SHAP explanations, and general recommendations in user-friendly language.

## Running the Application

Install dependencies:

```sh
pip install -r requirements.txt
```

Run the Streamlit app:

```sh
streamlit run SLEAP.py
```

## Current Development Status

- Improving robustness of the self-test page.
- Improving uploaded-dataset preprocessing and training workflow.
- Refining SHAP and reference-population explanations.
- Connecting chatbot responses more tightly to user-specific model outputs.
- Improving documentation, reproducibility, and testing.

## Intended Use and Limitations

SLEAP is intended for research, prototyping, and educational use. It is not a diagnostic medical device and should not be used as a substitute for professional medical evaluation, polysomnography, home sleep apnea testing, or clinical judgment.

Predictions generated by the platform depend on the input features, reference data, model training procedure, and selected threshold. Model outputs should be interpreted as decision-support information rather than definitive clinical conclusions.
